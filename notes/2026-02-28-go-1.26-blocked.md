# Go 1.26 upgrade blocked by syncthing dependency

## Status

`go.mod` intentionally kept at `go 1.25.0`. All other projects in the monorepo are on 1.26.

## Blocker

`go fix ./...` under Go 1.26 fails with:

```
# github.com/syncthing/syncthing/lib/api
syncthing@v1.30.0/lib/api/api_statics.go:38:25: undefined: auto.Assets
syncthing@v1.30.0/lib/api/api_statics.go:46:25: undefined: auto.Assets
```

The `auto.Assets` symbol is generated by syncthing's build toolchain (it requires running `go generate` with their asset pipeline before the package compiles). This works fine when building plop itself (the `noassets` build tag bypasses it), but `go fix` compiles all reachable packages in the module graph — including syncthing's `lib/api` — and hits the missing generated file.

## Resolution

Wait for a syncthing release that either:
- Ships pre-generated assets in the module, or
- Restructures the package so `lib/api` doesn't require generated code to compile

Then re-run `go fix ./...` and bump the `go` directive to 1.26.
