# Go 1.26 upgrade — RESOLVED (2026-03-01)

Migrated to Syncthing v2.0.14 which fixed both blockers. See `2026-03-01-syncthing-v2-migration.md`.

## Remaining step

Bump `go.mod` from `go 1.25.0` to `go 1.26.0` and run `go fix ./...`.
The `auto.Assets` blocker (from syncthing v1.30.0 `lib/api`) is gone since v2 restructured the package.

## Original blockers (historical)

### Blocker 1: `go fix` fails on syncthing v1.30.0 `lib/api`

`go fix ./...` compiled all reachable packages including syncthing's `lib/api` which required
generated `auto.Assets` — not present in the Go module. The `noassets` build tag only applied
to plop's own build, not `go fix`.

### Blocker 2: quic-go v0.52.0 runtime panic

Go 1.26 changed `crypto/tls` session ticket handling. quic-go v0.52.0 panicked during QUIC
TLS handshakes. Fixed in quic-go v0.53.0 (issue #5462), which syncthing v2.0.14 pulls as v0.56.0.
